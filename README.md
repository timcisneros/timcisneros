# Tim Cisneros

Full-stack software developer building applications that automate operational
work. I work across TypeScript, React, Vue.js, Node.js, SQL, and AWS, with an
emphasis on inspectable systems, reliable workflows, and tools people can use
on the job.

## Selected engineering work

- **[DSDebug](https://github.com/timcisneros/dsdebug-2023)** — Visual tooling
  for building and debugging DocuSign CLM workflows. It turns dense workflow
  exports into interactive graphs and reduced variable tracing from an
  afternoon of JSON inspection to minutes.
- **[Entity Visualization](https://github.com/timcisneros/entity-visualization)** —
  An anonymized public version of a professional React Flow organization chart
  for tracing ownership, parent and subsidiary relationships, distributions,
  and connected records across complex entity structures. The included dataset
  exercises 142 relationships across 10 charts and 73 entities.
- **[Self-Hosted YouTube](https://github.com/timcisneros/my-youtube)** — A
  private subscription frontend whose server proxies streams, thumbnails, and
  metadata so the browser never contacts Google. Its storage, queue, and worker
  dependencies are optional and swappable.
- **[Ticket System](https://github.com/timcisneros/ticket-system)** — A
  reference implementation for auditable agent work with durable ticket and
  run lifecycles, explicit authority, an evidence ledger, triage, replay, and
  independent verification.
- **[Waydaw](https://github.com/timcisneros/waydaw)** — A Linux/Wine
  compatibility harness for Ableton Live 12, Max for Live, and Windows plugins,
  backed by explicit recovery tooling and 59 non-mutating diagnostics.
- **[Action Plan Generator](https://github.com/timcisneros/seo-project)** — A
  focused TypeScript product that turns one YouTube link into a structured,
  time-boxed seven-day plan.

## Engineering approach

- Build across interfaces, APIs, data models, testing, and maintenance.
- Translate operational workflows into clear, inspectable software.
- Use AI-assisted development as an implementation tool, with explicit
  constraints, automated checks, and human verification.
- Document decisions and failure paths so systems remain understandable after
  the first successful run.
